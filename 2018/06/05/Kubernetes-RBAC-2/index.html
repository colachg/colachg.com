Do It Youself

动手试一试吧:

正式的生产环境中，除了运维人员可以直接和k8s集群交互外。其他测试和开发也想查看下集群的信息。比如:

  • 查看线上起了几个pod,
  • 每个pod的负载如何，
  • 每个pod的日志，
  • 线上API是否正常.

思考下，对应的RBAC怎么设置，并提供kubectl的config文件呢？

Kubernets API Groups

首先当然要知道有那些API分组了。

创建有命名空间限制权限的用户

然后确定要赋予角色哪些权限

kubectl create -f developer.yaml

生成kubectl的config文件

最后可以在home目录下的.kube文件夹下面看到config文件，具体其他需求可自行修改。
